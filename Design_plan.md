# Design Plan

메시지 웹앱 UI/UX 설계 문서 (카카오톡 유사 사용 흐름 기준).

## 1. 디자인 목표

- 모바일 메신저에 익숙한 사용 흐름 제공
- 빠른 정보 스캔(친구, 최근 대화, 미읽음)
- 한 손 사용 중심 인터랙션

## 2. 화면 구조

1. 친구 탭 (`/friends`)
- 상단: 검색창 + 옵션 버튼
- 본문: 즐겨찾기 친구 / 전체 친구
- 항목: 아바타, 이름, 상태메시지, 액션(프로필 보기)

2. 채팅 탭 (`/chats`)
- 상단: 검색 + 새 채팅 버튼
- 본문: 채팅 리스트
- 항목: 방 이름, 마지막 메시지, 시간, 미읽음 배지

3. 채팅방 (`/chats/:roomId`)
- 상단: 뒤로가기, 방 이름, 부가 메뉴
- 본문: 메시지 버블, 날짜 구분선
- 하단: 입력창 + 전송 버튼

4. 내 프로필 (`/profile`)
- 아바타, 이름, 상태메시지
- 편집 버튼
- 설정 영역(알림, 채팅 배경, 기타)

5. 하단 탭 네비게이션
- 친구 / 채팅 / 내 프로필
- 현재 탭 강조
- 채팅 탭에 미읽음 총합 배지 노출

## 3. 컴포넌트 설계

- `BottomTabBar`
- `FriendList`, `FriendItem`
- `ChatList`, `ChatListItem`
- `MessageList`, `MessageBubble`, `MessageInput`
- `ProfileCard`, `ProfileEditModal`
- `UnreadBadge`, `EmptyState`, `ErrorState`

## 4. 상태별 UI

1. 로딩 상태
- 스켈레톤 UI 사용

2. 빈 상태
- 친구 없음, 채팅 없음, 메시지 없음 문구 및 유도 액션

3. 오류 상태
- 재시도 버튼 제공
- 네트워크 오류와 서버 오류 문구 분리

## 5. 인터랙션 원칙

- 채팅방 진입 시 최신 메시지 위치로 이동
- 신규 메시지 도착 시 조건부 자동 스크롤
- 메시지 전송 즉시 Optimistic UI 반영
- 탭 전환 시 스크롤 위치 보존

## 6. 디자인 토큰(초안)

- 색상
  - Primary: `#FEE500`
  - Text Primary: `#111111`
  - Text Secondary: `#666666`
  - Border: `#E5E7EB`
  - Danger: `#DC2626`

- 타이포
  - 기본: `Pretendard`, `Noto Sans KR`, sans-serif
  - 본문: 14px/20px
  - 보조: 12px/16px

- 간격
  - 4, 8, 12, 16, 20, 24 단위

- 반경
  - 카드: 12px
  - 메시지 버블: 16px

## 7. 접근성/반응형

- 최소 터치 영역 44x44
- 색상 대비 WCAG AA 기준 준수
- 화면 폭 375px 기준 우선 설계, 768px 이상 2열 확장 고려

## 8. 마이크로카피(예시)

- 친구 없음: "친구를 추가해 대화를 시작해보세요."
- 채팅 없음: "아직 대화가 없습니다."
- 전송 실패: "메시지 전송에 실패했습니다. 다시 시도해주세요."

## 9. 검수 체크리스트

- 하단 탭에서 3개 메인 화면 접근 가능
- 채팅 리스트에서 미읽음/마지막 메시지 가독성 확인
- 채팅방 입력/전송/읽음 시각 피드백 확인
- 프로필 상세 및 편집 흐름 끊김 없는지 확인

## 10. 디자인 레퍼런스 (Figma)

- 적용 범위:
  - 채팅 목록 아이템 밀도/정보 위계
  - 채팅방 상단바와 입력 영역 배치
  - 메시지 버블 간격, 타임스탬프 처리, 미읽음 표시 방식
  - Material Design 3의 컴포넌트 톤을 참고하되 본 프로젝트 토큰과 IA를 우선

- 구현 원칙:
