# Phase 12: 브라우저 수동 테스트 & 즉시 수정 계획

**날짜:** 2026-02-20
**목적:** Phase 7-11 구현 결과를 Playwright 브라우저 도구로 직접 열어 확인하고, 발견된 문제를 즉시 수정

---

## 사전 준비

1. 서버 시작: `cd server && npx tsx src/index.ts` (포트 4000)
2. 클라이언트 시작: `cd client && npx vite` (포트 5173)
3. Playwright 브라우저: `browser_navigate` → `http://localhost:5173`

---

## 테스트 시나리오 (6단계)

### 1단계: 회원가입 & 로그인
- [ ] `/register` 스크린샷 → 로고, 레이아웃, 폼 확인
- [ ] 빈 폼 제출 → 에러 메시지 확인
- [ ] 비밀번호 5자 → "6자 이상" 에러
- [ ] 정상 회원가입 (testuser1@test.com / password123 / 테스트유저1) → /friends 이동
- [ ] 로그아웃 → /login 이동
- [ ] `/login` 스크린샷 → 로고 확인
- [ ] 잘못된 비밀번호 → 에러 메시지
- [ ] 정상 로그인 → /friends 이동
- [ ] 새로고침 → 토큰 유지 확인

### 2단계: 친구 목록
- [ ] `/friends` 스크린샷 → 빈 상태 UI
- [ ] 하단 탭 확인 (SVG 아이콘, 활성 indicator bar)
- [ ] "+" 버튼 → 이메일 입력 패널 표시
- [ ] 로그아웃 → 두 번째 계정 가입 (testuser2@test.com / 테스트유저2)
- [ ] 다시 첫 계정 로그인 → 친구 추가 (testuser2@test.com)
- [ ] 목록에 테스트유저2 표시 확인
- [ ] 즐겨찾기 토글 → 섹션 이동 확인
- [ ] 검색 입력 → 필터링 확인

### 3단계: 채팅 목록
- [ ] 하단 "채팅" 탭 → /chats 이동
- [ ] 빈 목록 스크린샷
- [ ] 친구 목록에서 1:1 채팅 개설 → /chats/:roomId 이동
- [ ] 뒤로가기 → 채팅 목록에 방 표시 (자동 타이틀 = 상대 이름)

### 4단계: 채팅방
- [ ] 채팅방 스크린샷 (헤더, 입력창)
- [ ] 메시지 입력 & 전송 → 내 메시지 (오른쪽, 노란색)
- [ ] 여러 메시지 → 시간 표시 확인
- [ ] 빈 메시지 전송 시도 → 차단 확인
- [ ] 뒤로가기 → 채팅 목록 마지막 메시지 미리보기

### 5단계: 프로필
- [ ] 하단 "프로필" 탭 → /profile 이동
- [ ] 프로필 정보 표시 (이름, 이메일, 상태메시지)
- [ ] "편집" → 이름/상태메시지 수정 → 저장 → 반영 확인
- [ ] 아바타 업로드 UI 확인 (Supabase key 미설정이면 에러 예상)
- [ ] 푸시 알림 섹션 확인
- [ ] 로그아웃 → /login 이동

### 6단계: 반응형 & 전체 UI
- [ ] 모바일 뷰포트 (375x812) → 각 페이지 스크린샷
- [ ] 데스크톱 뷰포트 (1280x800) → max-width 480px 센터링
- [ ] 탭 전환 시 활성 상태 시각적 구분 확인
- [ ] 콘솔 에러 점검 (`browser_console_messages`)

## 사용 도구

| 도구 | 용도 |
|------|------|
| `browser_navigate` | 페이지 이동 |
| `browser_take_screenshot` | 화면 캡처 |
| `browser_snapshot` | DOM 접근성 트리 |
| `browser_click` | 버튼/링크 클릭 |
| `browser_fill_form` / `browser_type` | 폼 입력 |
| `browser_press_key` | Enter 등 키 입력 |
| `browser_console_messages` | JS 에러 확인 |
| `browser_network_requests` | API 호출 상태 확인 |
| `browser_resize` | 뷰포트 변경 |

## 진행 방식

1. 각 단계 시작 전 스크린샷
2. 조작 후 결과 스크린샷
3. 문제 발견 시: 스크린샷 → 원인 파악 → 코드 수정 → 재확인
4. 전체 완료 후 최종 스크린샷 세트
